## Мнемонические системы <!-- легазизовать -->

Мнемонические системы являются одним из самых эффективных инструментов для запоминания информации, которые широко используются в нашей жизни.

Мнемоника – это искусство запоминания, которое базируется на принципах ассоциативного мышления и образной природы памяти. Она помогает человеку эффективно запоминать большое количество информации, например, иностранные слова, формулы, даты, имена и т.д. Каждый из нас может развить свою память, используя мнемонические техники.

Основные виды мнемонических техник которые я смог вывести:

- «История» – Метод выстраивания цепочки асоциацив в которой мы придумаваем историю в которой фигурируют запоминаемые предметы. Т.Бузана в своей книге "Скоростная память" приводил в котором, что бы запомнить предметы он придумал историю в которой он получал травмы из за тех продуктов которые было необходимо купить, некоторые продукты так же изменяли свое состояние в следствии травм получаемых от других продуктов[19].

- «Локализация» – метод, в котором мы ассоциируем информацию с местами и/или объектами, которые мы знаем хорошо и которые мы легко можем визуализировать. Например, вы можете изобразить вокзал, в котором вы видите сотни человек, с тем, что вас шквально обливают, чтобы запомнить множество иностранных слов, начинающихся с буквы "ш"[].

- «Акростих» – метод, в котором мы создаем мнемоническое устройство на основе первых букв слов, которые мы хотим запомнить. Например, вы можете запомнить все цвета радуги с помощью акростиха: "Каждый Охотник Желает Знать, Где Сидит Фазан". Каждое слово в этом стихе означает Первую букву цевета радуги: красный, оранжевый, желтый, зеленый, голубой, синий, фиолетовый[18].

- «Сокращения» (или «альфа-числовые») – метод, в котором мы используем первые буквы слов, чтобы создать сокращения, например, Украинская национальная ассоциация футбола (УНАФ) или для запоминания чисел, использующихся в математике и науке[].

Мнемоническая система лейтнера была основана Себастьяном Лейтнером (1919, Зальцбург — 1989) — немецкий комментатор и популяризатор науки.

Основной деятельностью Себастьяна Лейтнера были исследования.
Поначалу, основным предметом его исслудований были правовые и

(англ. Leitner System) для обучения по методике интервальных повторений[16].

Интервальные повторения (англ. spaced repetition) — техника удержания в памяти, заключающаяся в повторении запомненного учебного материала по определённым, постоянно возрастающим интервалам[20]. 

Этот принцип хорошо показал себя при изучении иностранных языков, однако этот принцип может использоватся для запоминания любой информации. Интервальные повторения не предполагают заучивания наизусть без понимания (но и не исключают его).

В классической системе лейтнера у запоминающего есть несколько коробок,
Каждая из коробок обозначает интервал времени после которого необходимо взять карточку из коробки о попытатся ответить на нее. Если запоминающему удалось ответить, карточка переходит в коробку с интервалом больше той в которой она находилась. Если запоминающий не может ответить на вопрос в карточке, он повторяет карточку и кладет ее в коробку с наименьшим интервалом из всех коробок.

- разработка приложений на основе нативных языков программирования. Этот метод снижает время и затраты на разработку, так как один раз написанный код может быть использован для разных систем. Однако, приложения, созданные с использованием гибридных технологий, могут иметь меньшую производительность, чем нативные приложения.

Для реализации моего проекта я выбрал React Native, 


React Native – это библиотека JavaScript от Facebook, которая позволяет разработчикам создавать настоящие мобильные приложения используя только JavaScript. React Native использует тот же язык программирования, что и веб, уменьшая порог вхождения для разработчиков.

React Native использует компонентный подход для разработки мобильных приложений. ### Разработка при помощи платформ-конструкторов

```
В современном мире, мобильные приложения являются важной частью жизни многих людей. Приложения позволяют быстро и удобно получать необходимую информацию, делать покупки, общаться с друзьями и многое другое. Разработка мобильных приложений требует определенных знаний и навыков программирования, поэтому создание своего приложения может оказаться достаточно сложной задачей для людей, не имеющих опыта в этой области. В такой ситуации платформ-конструкторы могут придти на выручку, предоставляя пользователю средства, которые позволяют разработать свое приложение без глубоких знаний кодирования.

Платформ-конструкторы для разработки мобильных приложений – это программные средства, которые предоставляют технический инструментарий для создания мобильных приложений, без написания кода. Существует большое количество платформ-конструкторов, которые позволяют создавать мобильные приложения на различных операционных системах, таких как Android и iOS.

Разработка мобильных приложений на платформах-конструкторах имеет множество преимуществ, среди которых:

1. Простота разработки. Платформы-конструкторы предоставляют удобный интерфейс визуального проектирования, с помощью которого можно создавать интерфейсы приложений без программирования.

2. Экономия времени и средств. Программирование мобильных приложений является дорогостоящим занятием, так как требует наличия квалифицированных специалистов. Разработка на платформах-конструкторах позволяет сократить время и затраты на создание мобильных приложений.

3. Универсальность. Платформы-конструкторы позволяют создавать мобильные приложения для различных операционных систем, таких как IOS и Android.

4. Простота доступа к инструментам разработки. Многие платформы-конструкторы имеют бесплатные пакеты инструментов разработки, которые позволяют создавать приложения любой сложности.

5. Наличие готовых компонентов. Платформы-конструкторы предоставляют широкий выбор готовых компонентов, таких как кнопки, текстовые поля, и многое другое, что существенно ускоряет разработку мобильных приложений.

Создание приложений на платформах-конструкторах имеет также свои недостатки. Один из них заключается в ограничении функционала приложения. Конструкторы позволяют создавать только те функции и элементы, которые предоставлены в самой платформе. Это означает, что приложение разрабатывается по заранее заданным параметрам. Также могут возникнуть проблемы с безопасностью приложения, так как платформы-конструкторы не всегда соответствуют современным нормам безопасности данных. 

Однако, все больше коммерческих приложений на рынке создано именно на платформах-конструкторах. Это связано с тем, что процесс разработки на конструкторах значительно упрощает создание минимального функционала существующего приложения для существующих платформ. Благодаря этому, коммерческим компаниям удается быстро создавать новые продукты и развивать бизнес без лишних затрат на программирование.

В заключении, можно сказать, что использование платформ-конструкторов для разработки мобильных приложений является важным инструментом для создания мобильных приложений. Однако, необходимо помнить, что это решение не идеально и может иметь свои недостатки. В любом случае, конечное решение должно быть основано на том, что наилучшим образом соответствует интересам пользователя и его проекту.
```

## Приложение 3
Формула Интервального повторения Y=2X+1, где Y — день, когда информация начнёт забываться, забывание идет постепенно и равняется Y дням, X — день последнего повторения после заучивания. Потенциал конечного интервала равен бесконечности. Некоторые слова и фразы могут запоминаться лучше, а другие хуже. Интервальные повторения чаще всего используют совместно с системой карточек.
